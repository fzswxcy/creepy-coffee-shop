# 微信小游戏美术资源使用指南

## 目录结构

```
assets/
├── ui/                    # 界面资源
│   ├── buttons/          # 按钮
│   ├── icons/            # 图标
│   ├── backgrounds/      # 背景
│   └── progress/         # 进度条
├── characters/           # 角色资源
│   ├── doctor/          # 医生顾客
│   ├── ghost/           # 幽灵顾客
│   ├── zombie/          # 僵尸顾客
│   └── vampire/         # 吸血鬼顾客
├── items/               # 物品资源
│   ├── coffee/          # 咖啡杯
│   ├── equipment/       # 设备
│   └── decoration/      # 装修元素
├── effects/             # 特效资源
│   ├── click/           # 点击特效
│   ├── upgrade/         # 升级特效
│   └── income/          # 收入特效
└── spritesheets/        # 精灵图
    ├── ui_spritesheet.png
    ├── ui_spritesheet.webp
    ├── ui_config.json
    ├── characters_spritesheet.png
    └── ...
```

## 命名规范

### 图片文件命名
- **语言**: 使用英文，小写字母
- **分隔符**: 使用下划线 `_`
- **格式**: `类别_名称_状态_序号.扩展名`

示例:
```
ui_button_start_normal.png
ui_button_start_pressed.png
character_doctor_walk_01.png
character_doctor_walk_02.png
item_coffee_latte.png
effect_click_gold_01.png
effect_click_gold_02.png
```

### 动画序列命名
```
character_ghost_idle_01.png
character_ghost_idle_02.png
character_ghost_idle_03.png
character_ghost_idle_04.png
```

### 状态命名
- `normal`: 正常状态
- `pressed`: 按下状态
- `disabled`: 禁用状态
- `hover`: 悬停状态
- `active`: 激活状态

## 图片格式选择

### 适合PNG格式的资源
- 需要透明背景的图片
- 有复杂透明度的图标
- 颜色数量较少的图片
- 按钮、图标等UI元素

### 适合WebP格式的资源
- 背景图片（无透明需求）
- 大尺寸图片
- 需要更好压缩率的资源
- 纹理图片

### 格式转换建议
```
原始PNG → pngquant压缩 → WebP转换
原始JPG → 直接转换为WebP
```

## 分辨率适配方案

### 多分辨率支持
```
资源名称                 设计尺寸    实际尺寸
ui_button_start.png     256×128    512×256 (@2x)
character_doctor.png    128×128    256×256 (@2x)
background_main.png     1080×1920  2160×3840 (@2x)
```

### 适配策略
1. **@2x资源制作**: 所有设计稿按2倍尺寸制作
2. **自适应缩放**: 小游戏引擎自动缩放
3. **九宫格拉伸**: UI元素使用九宫格技术
4. **动态加载**: 根据设备分辨率加载合适资源

## 精灵图使用指南

### 精灵图优势
1. **减少HTTP请求**: 合并多个图片为单个文件
2. **提高加载速度**: 一次性加载所有相关资源
3. **减少内存占用**: 减少纹理切换开销
4. **简化资源管理**: 统一配置文件管理

### 配置文件结构
```json
{
  "meta": {
    "image": "ui_spritesheet.png",
    "format": "RGBA8888",
    "size": { "w": 1024, "h": 1024 },
    "scale": 1
  },
  "frames": {
    "button_start": {
      "frame": { "x": 0, "y": 0, "w": 256, "h": 128 },
      "rotated": false,
      "trimmed": false,
      "spriteSourceSize": { "x": 0, "y": 0, "w": 256, "h": 128 },
      "sourceSize": { "w": 256, "h": 128 }
    }
  }
}
```

### 代码示例 - LayaAir
```typescript
// 加载精灵图配置
Laya.loader.load([
    { url: "assets/spritesheets/ui_spritesheet.png", type: Laya.Loader.IMAGE },
    { url: "assets/spritesheets/ui_config.json", type: Laya.Loader.JSON }
], Laya.Handler.create(this, this.onAssetsLoaded));

// 使用精灵图
private onAssetsLoaded(): void {
    // 创建精灵
    let sprite = new Laya.Sprite();
    let texture = Laya.loader.getRes("assets/spritesheets/ui_spritesheet.png");
    
    // 绘制精灵图中的特定区域
    sprite.graphics.drawTexture(texture, 
        0, 0, 256, 128,  // 绘制区域
        0, 0, 256, 128   // 精灵图源区域
    );
}
```

### 代码示例 - Cocos Creator
```javascript
// 加载精灵图
cc.resources.load("assets/spritesheets/ui_spritesheet", cc.SpriteFrame, (err, spriteFrame) => {
    if (err) {
        console.error(err);
        return;
    }
    
    // 创建精灵
    let sprite = this.node.getComponent(cc.Sprite);
    sprite.spriteFrame = spriteFrame;
    
    // 设置精灵图矩形
    sprite.spriteFrame.setRect(cc.rect(0, 0, 256, 128));
});
```

## 性能优化建议

### 加载优化
1. **预加载关键资源**: 游戏启动时预加载UI和核心角色
2. **分帧加载**: 非关键资源分帧加载，避免卡顿
3. **按需加载**: 场景切换时加载对应资源
4. **缓存策略**: 合理设置资源缓存时间

### 内存优化
1. **及时销毁**: 离开场景时销毁不再使用的资源
2. **纹理管理**: 使用纹理图集减少纹理切换
3. **对象池**: 复用游戏对象减少GC压力
4. **分辨率适配**: 根据设备内存加载合适分辨率

### 渲染优化
1. **批处理**: 合并相同材质的渲染调用
2. **减少Draw Call**: 优化渲染顺序和合并
3. **帧率控制**: 动画帧率控制在30fps以内
4. **避免过度绘制**: 简化复杂UI层次

## 动画制作规范

### 帧动画规范
1. **帧率**: 控制在24-30fps之间
2. **帧数**: 循环动画8-12帧，特效动画6-15帧
3. **尺寸**: 动画序列图片尺寸保持一致
4. **命名**: 序号从01开始，保持连续

### 骨骼动画建议
1. **优先使用帧动画**: 微信小游戏对骨骼动画支持有限
2. **简单骨骼动画**: 如必须使用，保持骨骼数量精简
3. **导出格式**: 使用引擎支持的格式（如Laya的sk格式）

4. **设备检查**: 在不同设备上检查颜色显示一致性

## 异常处理

### 资源加载失败
```javascript
// 资源加载错误处理
Laya.loader.load(urls, Laya.Handler.create(this, this.onSuccess), 
    Laya.Handler.create(this, this.onProgress),
    Laya.Handler.create(this, this.onError)
);

private onError(err: any): void {
    console.error("资源加载失败:", err);
    // 加载备用资源或显示错误提示
    this.loadFallbackResource();
}
```

### 内存警告处理
```javascript
// 内存警告处理
if (memoryWarning) {
    // 释放非必要资源
    this.releaseUnusedResources();
    // 降低渲染质量
    this.lowerRenderQuality();
}
```
