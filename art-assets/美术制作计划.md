## 项目概述
- **项目类型**: 微信小游戏 (咖啡店微恐主题)

## 制作规范

### 颜色规范
- **主色调**: #2C3E50 (深蓝灰)
- **强调色**: #FF6B6B (警告红)
- **辅助色**: #4ECDC4 (青绿色)
- **文字色**: #FFFFFF (白色) / #2C3E50 (深蓝灰)
- **背景色**: #1A252F (更深灰)

### 分辨率适配策略
1. **基础分辨率**: 1080x1920px (用于设计稿)
2. **适配方案**: 
   - 2倍图: 2160x3840px (实际制作)
   - 压缩到: 540x960px (低分辨率设备)
   - 使用九宫格技术适配不同屏幕

### 文件格式与优化
1. **静态资源**: PNG-8 (256色) + 透明通道
2. **动态资源**: APNG 或 Sprite Sheet
3. **背景图**: 使用WebP格式 (压缩率更高)
4. **精灵图**: 合并同类资源减少HTTP请求

### 性能优化策略
1. **图片压缩**:
   - PNG: 使用pngquant压缩
   - WebP: 设置quality=80
   - 移除EXIF信息

2. **内存优化**:
   - 按需加载资源
   - 及时销毁无用纹理
   - 使用纹理图集

3. **动画优化**:
   - 帧率限制在30fps以内
   - 减少关键帧数量
   - 使用CSS动画代替JS动画

## 资源使用指南

### 命名规范
```
ui/button_start_normal.png
ui/button_start_pressed.png
characters/doctor_walk_01.png
items/coffee_latte.png
effects/click_gold_01.png
```

### 导入指南
1. UI资源放在 `/assets/ui/`
2. 角色资源放在 `/assets/characters/`
3. 物品资源放在 `/assets/items/`
4. 特效资源放在 `/assets/effects/`
5. 精灵图放在 `/assets/spritesheets/`

### 性能建议
1. **图片加载**: 使用预加载机制
2. **内存管理**: 定时清理缓存
3. **动画优化**: 使用requestAnimationFrame
4. **适配方案**: 使用动态分辨率适配

## 成本控制措施
1. **重用资源**: 同一角色不同颜色变体
2. **模块化设计**: 组件化设计减少重复工作
3. **自动化工具**: 使用脚本批量处理图片
4. **开源素材**: 适当使用免费素材修改

---

**美术师备注**: 
- 所有设计需优先考虑小游戏性能限制
- 恐怖元素可爱化是关键，避免真实恐怖感
- 颜色搭配需符合微恐主题但保持视觉舒适度
- 动画帧数控制在合理范围，避免过度复杂
