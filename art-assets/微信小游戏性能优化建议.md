# 微信小游戏美术资源性能优化建议

## 一、微信小游戏平台特性

### 技术限制
- **包体大小**: ≤ 8MB (首次加载包)
- **内存限制**: ≤ 100MB (不同设备有差异)
- **CPU限制**: 低端机性能有限
- **GPU限制**: 部分设备不支持高级特性
- **网络限制**: 小游戏加载速度敏感

### 美术资源直接影响
1. **包体大小**: 直接影响小游戏审核通过率
2. **加载速度**: 影响用户体验和留存率
3. **运行性能**: 影响游戏流畅度和稳定性
4. **内存占用**: 影响多任务切换和长时运行

## 二、图片资源优化策略

### 1. 图片格式选择
#### PNG格式
- **适用场景**: 需要透明通道的UI元素、图标
- **优化要点**:
  - 使用PNG-8 (256色) 代替PNG-24
  - 移除不必要的元数据 (EXIF, ICC Profile)
  - 使用pngquant等工具深度压缩
  - 避免使用过多的渐变和阴影

#### JPEG格式
- **适用场景**: 背景图片、无透明需求的大图
- **优化要点**:
  - 设置合适的质量 (60-80%)
  - 使用渐进式JPEG
  - 避免重复压缩
  - 移除EXIF信息

#### WebP格式
- **适用场景**: 所有静态图片 (微信小游戏支持)
- **优势**:
  - 相比PNG小25-35%
  - 相比JPEG小25-34%
  - 支持透明通道
  - 压缩率可调
- **注意**: 需要兼容性检查，提供fallback

### 2. 图片尺寸优化
#### 尺寸策略
- **@2x设计**: 按2倍尺寸设计，适配高DPI设备
- **动态缩放**: 运行时根据设备缩放
- **九宫格拉伸**: UI元素使用九宫格减少图片数量

#### 最大尺寸限制
| 资源类型 | 最大宽度 | 最大高度 | 备注 |
|----------|----------|----------|------|
| 背景图 | 2048px | 2048px | 避免超大背景 |
| UI元素 | 512px | 512px | 按钮、图标等 |
| 角色 | 256px | 256px | 小尺寸角色 |
| 特效 | 128px | 128px | 动画特效 |

## 三、内存使用优化

### 1. 纹理内存优化
#### 减少纹理数量
- **精灵图**: 合并小图片为精灵图
- **纹理图集**: 游戏引擎的纹理打包
- **共享纹理**: 相同材质的对象共享纹理

#### 纹理格式选择
- **RGBA8888**: 高质量，内存占用大
- **RGBA4444**: 中等质量，内存减半
- **RGB565**: 无透明通道，内存更小
- **ETC1**: Android设备专用压缩纹理

#### 分帧加载
```javascript
// 避免一次性加载大量资源
function loadInFrames(resources, callback) {
    let index = 0;
    
    function loadNext() {
        if (index >= resources.length) {
            callback();
            return;
        }
        
        // 每帧加载一个资源
        loadResource(resources[index], () => {
            index++;
            setTimeout(loadNext, 0); // 下一帧继续
        });
    }
    
    loadNext();
}
```

### 3. 对象池技术
```javascript
// 对象池示例
class ObjectPool {
    constructor(createFunc, maxSize = 100) {
        this.pool = [];
        this.createFunc = createFunc;
        this.maxSize = maxSize;
    }
    
    get() {
        if (this.pool.length > 0) {
            return this.pool.pop();
        }
        return this.createFunc();
    }
    
    release(obj) {
        if (this.pool.length < this.maxSize) {
            obj.reset(); // 重置对象状态
            this.pool.push(obj);
        } else {
            // 销毁对象
            obj.destroy();
        }
    }
}

// 使用对象池
const bulletPool = new ObjectPool(() => new Bullet(), 50);
const bullet = bulletPool.get();
// ... 使用后
bulletPool.release(bullet);
```

## 四、渲染性能优化

### 1. 减少Draw Call
#### 合并渲染批次
- **相同材质合并**: 尽量使用相同shader和纹理
- **静态批处理**: 不动的物体合并渲染
- **动态批处理**: 小物体自动批处理

#### 渲染顺序优化
```javascript
// 优化渲染顺序
function optimizeRenderOrder(gameObjects) {
    // 按材质分组
    const groups = {};
    
    gameObjects.forEach(obj => {
        const key = `${obj.material}_${obj.texture}`;
        if (!groups[key]) groups[key] = [];
        groups[key].push(obj);
    });
    
    // 按组渲染
    Object.values(groups).forEach(group => {
        renderGroup(group);
    });
}
```

### 2. 动画性能优化
#### 帧动画优化
- **帧率限制**: 控制在24-30fps
- **帧数优化**: 循环动画8-12帧足够
- **尺寸统一**: 动画序列图片尺寸一致
- **预加载**: 动画资源提前加载

#### 骨骼动画优化
- **骨骼数量**: 尽量少于20个骨骼
- **网格顶点**: 尽量少于1000个顶点
- **动画数据**: 压缩动画关键帧数据
- **GPU加速**: 使用GPU处理动画计算

### 3. 特效优化
#### 粒子特效
- **粒子数量**: 控制在50-100个以内
- **发射频率**: 适当降低发射频率
- **生命周期**: 缩短粒子生命周期
- **材质简化**: 使用简单材质

#### 屏幕特效
- **避免全屏特效**: 减少全屏后处理
- **简化shader**: 使用简单shader
- **分辨率适配**: 根据设备降低分辨率

## 五、包体大小控制

### 1. 资源大小监控
#### 实时监控脚本
```javascript
// 资源大小监控
class SizeMonitor {
    constructor(maxSize = 8 * 1024 * 1024) { // 8MB
        this.maxSize = maxSize;
        this.currentSize = 0;
        this.resources = new Map();
    }
    
    addResource(name, size) {
        this.resources.set(name, size);
        this.currentSize += size;
        
        if (this.currentSize > this.maxSize) {
            console.warn(`包体大小超标: ${this.currentSize} > ${this.maxSize}`);
            this.suggestOptimizations();
        }
    }
    
    suggestOptimizations() {
        const suggestions = [];
        
        // 分析大文件
        const largeFiles = Array.from(this.resources.entries())
            .filter(([name, size]) => size > 100 * 1024) // >100KB
            .sort((a, b) => b[1] - a[1]);
        
        if (largeFiles.length > 0) {
            suggestions.push('大文件优化建议:');
            largeFiles.forEach(([name, size]) => {
                suggestions.push(`  ${name}: ${Math.round(size/1024)}KB`);
            });
        }
        
        return suggestions;
    }
}
```

## 六、设备兼容性

### 1. 分辨率适配
```javascript
// 动态分辨率适配
class ResolutionAdapter {
    static getOptimalResolution() {
        const systemInfo = wx.getSystemInfoSync();
        
        // 根据设备性能选择分辨率
        const performance = systemInfo.benchmarkLevel || 0;
        
        switch (performance) {
            case 0: // 低端机
                return { scale: 0.5, quality: 'low' };
            case 1: // 中端机
                return { scale: 0.75, quality: 'medium' };
            case 2: // 高端机
                return { scale: 1.0, quality: 'high' };
            default:
                return { scale: 0.75, quality: 'medium' };
        }
    }
    
    static loadAssets(quality) {
        const assets = {
            low: 'assets/low/',
            medium: 'assets/medium/',
            high: 'assets/high/'
        };
        
        return assets[quality] || assets.medium;
    }
}
```

### 2. 内存警告处理
```javascript
// 内存警告监听
wx.onMemoryWarning((res) => {
    console.log('内存警告:', res.level);
    
    switch (res.level) {
        case 5: // 紧急
            emergencyMemoryRelease();
            break;
        case 10: // 严重
            aggressiveMemoryRelease();
            break;
        case 15: // 一般
            normalMemoryRelease();
            break;
    }
});

function emergencyMemoryRelease() {
    // 紧急释放内存
    releaseAllNonEssentialResources();
    lowerRenderQualityToMinimum();
    stopAllAnimations();
}
```

### 3. 性能监控
```javascript
// 性能监控
class PerformanceMonitor {
    constructor() {
        this.fps = 60;
        this.memory = 0;
        this.drawCalls = 0;
        this.startTime = Date.now();
        
        this.monitor();
    }
    
    monitor() {
        setInterval(() => {
            this.updateStats();
            this.checkThresholds();
        }, 1000); // 每秒更新
    }
    
    updateStats() {
        // 获取当前性能数据
        this.fps = this.calculateFPS();
        this.memory = this.getMemoryUsage();
        this.drawCalls = this.getDrawCalls();
    }
    
    checkThresholds() {
        if (this.fps < 20) {
            console.warn('帧率过低:', this.fps);
            this.suggestOptimizations();
        }
        
        if (this.memory > 80 * 1024 * 1024) { // 80MB
            console.warn('内存使用过高:', this.memory);
            this.suggestMemoryRelease();
        }
    }
}
```
