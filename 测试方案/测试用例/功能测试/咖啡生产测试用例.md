# 咖啡生产功能测试用例

## 测试概述
测试咖啡生产相关的所有功能，包括设备操作、产出计算、升级效果等。

## 测试用例列表

### TC-FUNC-001: 基础咖啡机点击生产
- **测试场景**: 玩家点击基础咖啡机生产咖啡
- **前置条件**: 
  1. 游戏已正常启动
  2. 玩家有足够的咖啡豆
  3. 咖啡机处于可用状态
- **测试步骤**:
  1. 进入咖啡厅场景
  2. 点击基础咖啡机图标
  3. 观察咖啡生产动画
  4. 查看咖啡库存变化
- **预期结果**:
  1. 咖啡机有点击反馈（视觉/音效）
  2. 播放咖啡生产动画
  3. 咖啡库存增加1杯基础咖啡
  4. 咖啡豆库存相应减少
- **优先级**: P0
- **测试类型**: 手动测试 + 自动化测试

## 自动化测试脚本示例

```javascript
// 咖啡生产自动化测试脚本
const coffeeProductionTests = {
  testBasicProduction: async () => {
    // 模拟点击咖啡机
    await clickElement('coffee-machine-btn');
    
    // 验证库存变化
    const beforeCount = await getInventoryCount('basic-coffee');
    await waitForAnimation();
    const afterCount = await getInventoryCount('basic-coffee');
    
    assert.equal(afterCount - beforeCount, 1, '咖啡库存应增加1');
    
    // 验证咖啡豆消耗
    const beforeBeans = await getInventoryCount('coffee-beans');
    const afterBeans = await getInventoryCount('coffee-beans');
    assert.equal(beforeBeans - afterBeans, 1, '咖啡豆应减少1');
  },
  
  testInsufficientBeans: async () => {
    // 设置咖啡豆为0
    await setInventoryCount('coffee-beans', 0);
    
    // 尝试生产
    await clickElement('coffee-machine-btn');
    
    // 验证提示信息
    const message = await getToastMessage();
    assert.include(message, '咖啡豆不足', '应显示咖啡豆不足提示');
    
    // 验证库存未变化
    const coffeeCount = await getInventoryCount('basic-coffee');
    assert.equal(coffeeCount, 0, '咖啡库存不应变化');
  },
  
  testUpgradeEffect: async () => {
    // 记录升级前生产效率
    const beforeTime = await measureProductionTime();
    
    // 执行升级
    await upgradeCoffeeMachine();
    
    // 记录升级后生产效率
    const afterTime = await measureProductionTime();
    
    // 验证效率提升
    assert.isBelow(afterTime, beforeTime, '升级后生产效率应提高');
    
    // 验证等级提升
    const level = await getMachineLevel('coffee-machine');
    assert.equal(level, 2, '咖啡机等级应提升到2级');
  }
};

module.exports = coffeeProductionTests;
```

## 性能测试指标

### 生产响应时间
- **目标值**: <100ms
- **测量方法**: 从点击到动画开始的时间差
- **测试设备**: 高端、中端、低端各测10次取平均

### 生产动画性能
- **目标FPS**: >30fps
- **测量方法**: 动画播放期间的帧率监控
- **通过标准**: 无卡顿，动画流畅

### 内存使用
- **基线内存**: 记录生产前的内存使用
- **生产时内存**: 记录生产动画期间的内存峰值
- **增量要求**: <5MB内存增加
- **恢复能力**: 生产结束后内存应回落

## 测试总结要点

### 关键成功指标
1. **功能完整性**: 所有生产相关功能正常工作
2. **数据准确性**: 库存计算100%准确
3. **性能达标**: 生产响应<100ms，动画流畅
4. **用户体验**: 操作反馈及时，提示清晰

### 风险关注点
1. **并发生产**: 多机器同时生产可能引发数据同步问题
2. **离线计算**: 离线生产算法需要精确验证
3. **数值平衡**: 生产效率与消耗需要合理平衡
4. **极端情况**: 边界值和异常输入处理

### 优化建议
1. **性能优化**: 考虑生产动画的优化方案
2. **用户体验**: 添加生产进度提示
3. **数据安全**: 生产数据需要服务端验证
4. **扩展性**: 为未来更多生产设备预留接口
