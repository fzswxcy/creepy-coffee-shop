   - 支持本地文件和网络URL

   - `--to` 参数必须指定用户 openid
   - `--delete-after-run` 对一次性任务很重要

3. **系统架构理解**：
   - QQ Bot 使用 openid 作为用户标识
   - 定时任务运行在 isolated 会话中
   - 消息发送需要指定 channel 参数

## 2026-02-28 - QQ Bot 定时提醒深度调试

### 🔍 发现的重要问题
在进行QQ Bot定时提醒测试时，发现关键问题：

1. **消息发送机制故障**：
   - 定时任务执行成功但消息未送达用户
   - 直接使用`message`工具发送失败（错误："to required"）
   - QQ Bot状态显示正常但消息传递链路可能有问题

2. **定时任务类型理解**：
   - `systemEvent`类型：用于main会话，直接触发系统事件
   - `agentTurn`类型：用于isolated会话，AI助手执行任务
   - 使用`--announce`参数确保结果发送到指定通道

### 🔧 创建的测试任务
为了排查问题，创建了多个测试任务：

1. **系统事件测试** (main会话)
   - 使用`--system-event`参数
   - 测试系统事件触发机制

2. **简单定时测试** (isolated会话)
   - 使用`--announce`参数
   - 测试独立会话直接消息发送

### 💡 重要技术经验
1. **关键参数组合**：
   ```bash
   openclaw cron add \
     --name "任务名称" \
     --at "时间" \
     --session isolated \
     --message "内容" \
     --announce \
     --channel qqbot \
     --to "用户openid" \
     --delete-after-run
   ```

2. **问题排查流程**：
   - 检查定时任务执行状态 (`openclaw cron runs --id <jobId>`)
   - 查看会话历史 (`sessions_history`)
   - 检查QQ Bot配置状态 (`openclaw status`)
