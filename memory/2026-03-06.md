## 💡 重要技术决策

### 1. 断点恢复系统架构
**决策**: 采用 `project_state.md` + token监控 + 自动 `/reset` 方案
**理由**: 
- 简单可靠，不依赖复杂数据库
- 实时状态管理，最小token占用
- 自动断点触发，无需人工干预
- 零中断体验，用户无感知

### 2. 游戏接口设计
**决策**: 单例模式 + 事件驱动 + TypeScript强类型
**理由**:
- 统一接口，降低系统复杂度
- 类型安全，减少运行时错误
- 事件驱动，实现松耦合架构
- 易于测试，完整测试覆盖

### 3. 微信平台适配策略
**决策**: 适配器模式 + 模拟环境 + 自动检测
**理由**:
- 平台无关代码，易于移植到其他平台
- 开发友好，非微信环境可正常开发
- 自动环境检测，运行时自适应
- 完整功能覆盖，支持所有微信特性
