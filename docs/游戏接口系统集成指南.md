# 🎮 游戏接口系统集成指南

## 📋 概述
`GameInterface.ts` 是连接UI系统和游戏逻辑系统的核心桥梁。它采用单例模式，提供统一的事件传递和数据同步机制。

## 🔧 核心文件

### 1. GameInterface.ts
**位置**: `assets/scripts/core/GameInterface.ts`
**功能**: 全局唯一的游戏接口实例，连接所有子系统

### 2. GameInterfaceTest.ts
**位置**: `assets/scripts/core/GameInterfaceTest.ts`
**功能**: 完整的接口测试系统，验证所有连接功能

### 3. MainSceneUIController.ts
**位置**: `assets/scripts/ui/MainSceneUIController.ts`
**功能**: 主场景UI控制器，与GameInterface交互

### 4. MainGameLoop.ts
**位置**: `assets/scripts/game/MainGameLoop.ts`
**功能**: 主游戏循环，提供游戏状态和控制接口

### 步骤1：初始化游戏接口
在应用启动时初始化GameInterface：

```typescript

import { gameInterface } from './assets/scripts/core/GameInterface';
import { MainSceneUIController } from './assets/scripts/ui/MainSceneUIController';

export function initializeGame() {
    // 创建UI控制器实例

    gameInterface.initialize(uiController);

}
```

### 步骤3：配置管理器回调
确保所有管理器系统提供事件监听接口：

```typescript
// 示例：EconomyManager添加事件接口
export class EconomyManager {
    private money: number = 0;
    private moneyChangeCallbacks: ((newAmount: number) => void)[] = [];
    
    public onMoneyChanged(callback: (newAmount: number) => void): void {
        this.moneyChangeCallbacks.push(callback);
    }
    
    private notifyMoneyChanged(newAmount: number): void {
        this.moneyChangeCallbacks.forEach(callback => callback(newAmount));
    }
}
```

## 🔄 事件流程图

```
用户操作
    ↓
UI控制器 (MainSceneUIController)
    ↓
游戏接口 (GameInterface)
    ↓
相应管理器系统 (经济/顾客/生产等)
    ↓
游戏循环 (MainGameLoop)
    ↓
UI更新 (通过事件回调)
```

## 📊 数据流

### 正向数据流（用户操作 → 游戏逻辑）
1. 用户点击UI按钮
2. UI控制器调用相应方法
3. GameInterface转发到对应的管理器
4. 管理器处理逻辑，更新游戏状态
5. GameInterface通知相关系统

### 反向数据流（游戏状态 → UI更新）
1. 游戏状态发生变化
2. GameInterface监听状态变化
3. 调用UI控制器的更新方法
4. UI控制器更新界面显示

## 🚨 常见问题

### 问题1：接口未初始化
**症状**: `gameInterface.isReady()` 返回false
**解决方案**: 确保在应用启动时调用 `gameInterface.initialize(uiController)`

### 问题2：事件不响应
**症状**: UI变化后游戏状态未更新
**解决方案**: 
1. 检查事件监听器是否正确设置
2. 验证回调函数签名匹配
3. 检查管理器事件触发机制

### 问题3：数据不同步
**症状**: UI显示与游戏数据不一致
**解决方案**:
1. 检查数据更新时机
2. 验证事件传递路径
3. 添加调试日志

## 📈 性能优化建议

1. **事件节流**: 高频事件（如时间更新）使用节流
2. **批量更新**: 多个UI更新合并为一次
3. **惰性计算**: 复杂的统计数据延迟计算
4. **内存管理**: 及时清理不再使用的监听器
