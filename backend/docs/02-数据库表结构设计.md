# 数据库表结构设计

## 数据库设计原则

### 1. 成本控制优先
- **最小化字段**: 每个字段都有明确用途
- **数据类型优化**: 使用合适的数据类型减少存储
- **索引策略**: 只为高频查询字段创建索引
- **分表设计**: 按业务模块分表，便于扩展

### 2. 性能优化
- **读写分离**: 高频写操作与读操作分离
- **数据分区**: 按时间或用户ID分区
- **缓存策略**: 热点数据使用内存缓存
- **批量操作**: 支持批量读写减少API调用

### 3. 扩展性设计
- **版本兼容**: 数据结构支持版本升级
- **字段预留**: 关键表预留扩展字段
- **分库分表**: 预留分库分表方案

## 集合设计优化

### 1. 集合命名规范
- **小写字母**: 全部使用小写
- **下划线分隔**: 使用下划线分隔单词
- **业务前缀**: 按业务模块分组
- **示例**: `users`, `game_progress`, `orders`, `ad_records`, `leaderboard`, `friend_relations`

### 2. 分表策略
- **按时间分表**: 订单表按月分表（orders_202503）
- **按用户分表**: 用户数据量过大时按用户ID哈希分表
- **按业务分表**: 不同业务模块使用不同集合

## 数据库访问模式

### 1. 读多写少模式
```javascript
// 示例：读取用户数据
async function getUserData(openid) {
  // 1. 尝试从缓存读取
  const cached = await cache.get(`user:${openid}`);
  if (cached) return cached;
  
  // 2. 从数据库读取
  const user = await db.collection('users').doc(openid).get();
  
  // 3. 更新缓存
  await cache.set(`user:${openid}`, user.data, 300); // 缓存5分钟
  
  return user.data;
}
```

### 2. 批量写入模式
```javascript
// 示例：批量更新订单状态
async function completeOrders(orderIds) {
  const db = wx.cloud.database();
  const _ = db.command;
  
  return await db.collection('orders').where({
    _id: _.in(orderIds)
  }).update({
    data: {
      status: 'completed',
      updatedAt: db.serverDate()
    }
  });
}
```

### 3. 事务处理模式
```javascript
// 示例：咖啡制作完成后的多表更新
async function completeCoffeeMaking(orderId, userId) {
  const transaction = await db.startTransaction();
  
  try {
    // 1. 更新订单状态
    await transaction.collection('orders').doc(orderId).update({
      status: 'completed',
      completedAt: new Date()
    });
    
    // 2. 更新用户金币
    await transaction.collection('users').doc(userId).update({
      'gameData.coins': _.inc(15),
      'statistics.totalOrders': _.inc(1),
      'statistics.totalRevenue': _.inc(15)
    });
    
    // 3. 提交事务
    await transaction.commit();
    
    return { success: true };
  } catch (error) {
    // 回滚事务
    await transaction.rollback();
    throw error;
  }
}
```

## 性能优化建议

### 1. 查询优化
- **使用投影**: 只查询需要的字段
- **限制结果集**: 使用limit限制返回数量
- **避免全表扫描**: 使用索引优化查询
- **分页查询**: 大数据集使用分页查询

### 2. 写入优化
- **批量写入**: 批量操作减少API调用
- **异步写入**: 非关键数据异步写入
- **延迟更新**: 高频更新数据延迟合并更新

### 3. 索引优化
- **控制索引数量**: 每个集合不超过5个索引
- **复合索引**: 常用查询组合使用复合索引
- **定期维护**: 定期分析索引使用情况

## 迁移和升级策略

### 1. 数据结构迁移
```javascript
// scripts/migrate-database.js
async function migrateUserSchema(fromVersion, toVersion) {
  // 从旧版本迁移到新版本
  const users = await db.collection('users').where({
    version: fromVersion
  }).get();
  
  for (const user of users.data) {
    // 执行迁移逻辑
    const migrated = migrateUserData(user, fromVersion, toVersion);
    
    await db.collection('users').doc(user._id).update({
      data: migrated,
      version: toVersion
    });
  }
}
```

### 2. 分表迁移
```javascript
// 将旧订单表迁移到新的分表结构
async function migrateOrdersToMonthlyTables() {
  const orders = await db.collection('orders').get();
  
  for (const order of orders.data) {
    const month = getMonthFromDate(order.createdAt);
    const targetCollection = `orders_${month}`;
    
    await db.collection(targetCollection).add({
      data: order
    });
  }
}
```
