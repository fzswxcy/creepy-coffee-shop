# 《微恐咖啡厅》微信小游戏UI/UX设计文档

## 项目概述
一款融合微恐元素的咖啡厅经营类微信小游戏，需要在简约易用的前提下营造适度的恐怖氛围，突出变现入口，同时确保性能优化。

## 一、设计风格与原则

### 1.1 核心设计原则
- **简约直观**：操作流程不超过3步点击
- **微恐氛围**：轻度恐怖元素，不吓人（PG-13级）
- **变现优先**：关键变现入口置于视觉焦点区
- **性能优化**：资源控制在微信小游戏限制内

### 1.2 美术风格
- **画面风格**：手绘卡通+暗色系微恐元素
- **分辨率**：适配微信小游戏（建议750×1334px）
- **色彩深度**：避免过多渐变，使用纯色+纹理

## 三、核心UI组件设计

### 3.1 按钮组件
```
1. 主要按钮 (Primary Button)
   - 尺寸: 200×60px
   - 颜色: #FF6B6B (警告红) + 微光效果
   - 状态: 正常/按下/禁用
   - 点击反馈: 轻微缩放+粒子效果

2. 次要按钮 (Secondary Button)
   - 尺寸: 180×50px
   - 颜色: #4ECDC4 (青绿色)
   - 边框: 2px #FFE66D (黄色)

3. 广告按钮 (Ad Button)
   - 尺寸: 160×50px
   - 颜色: #FFE66D (亮黄色) + 闪烁动画
   - 图标: 📺 + "看广告"
```

### 3.2 卡片组件
```
商品卡片
├─顶部: 商品图片 (128×128px)
├─中部: 商品名称 + 简短描述
├─底部: 价格 + 购买按钮
└─角标: NEW/HOT/限时 (微恐元素)

顾客卡片
├─左侧: 角色头像 (圆形80px)
├─中部: 名称 + 满意度进度条
├─右侧: 等待时间/需求图标
└─点击: 展开详情动画
```

### 3.3 进度指示器
```
1. 经验进度条
   - 背景: #2C3E50 (深蓝灰)
   - 填充: #FF6B6B (红色渐变)
   - 动画: 缓慢填充效果

2. 制作进度环
   - 外环: 幽灵绿色发光
   - 内环: 进度百分比
   - 完成: 幽灵飞出动画
```

### 3.4 变现入口组件
```
1. 钻石/金币展示区
   - 位置: 屏幕右上角固定
   - 样式: 图标+数字+[+]按钮
   - 点击+: 弹出充值页面

2. 每日奖励弹窗
   - 时间: 首次登录显示
   - 内容: 7日连续奖励
   - 激励: 第7天双倍奖励

3. 看广告按钮组
   - 位置: 关键操作旁
   - 文案: "看广告加速" / "免费获得"
   - 频率: 每操作最多1次广告
```

## 五、美术资源建议清单

### 5.1 必须资源 (优先级1)
```
1. 背景素材
   - 咖啡厅主场景 ×1 (白天/夜晚切换)
   - 设备底图 (咖啡机/收银台等) ×5
   - 地板纹理 ×2 (木地板/诡异地板)

2. UI元素
   - 按钮状态 (正常/按下/禁用) ×3套
   - 图标集 (100个常用图标)
   - 进度条组件 ×4种

3. 角色资源
   - 顾客立绘 ×8 (微恐风格)
   - 员工角色 ×2 (可选)
   - 角色表情 ×3/角色
```

### 5.2 可选资源 (优先级2)
```
1. 动画效果
   - 幽灵飘动动画 ×3
   - 制作完成特效 ×2
   - 金币获取特效 ×1

2. 装饰元素
   - 万圣节装饰 ×5 (蝙蝠/蜘蛛网等)
   - 季节限定装饰 ×4套

3. 音效资源
   - 背景音乐 ×2 (轻松/微恐)
   - 音效 ×15 (点击/完成/恐怖等)
```

### 5.3 性能优化建议
```
1. 图片格式
   - UI元素: PNG (透明需求)
   - 背景: JPG (压缩质量85%)
   - 图标: 使用雪碧图打包

2. 尺寸限制
   - 单图最大: 512×512px
   - 总资源: < 10MB (微信限制)
   - 按需加载: 分场景加载资源

3. 动画优化
   - 使用CSS动画代替GIF
   - 骨骼动画优先于序列帧
   - 限制同时播放动画数量
```

## 六、变现系统设计

### 6.1 变现入口布局
```
1. 主界面 (3个入口)
   - 右上角: 钻石充值 (固定)
   - 底部栏: 广告收益 (视觉突出)
   - 离线收益: 看广告双倍

2. 子界面 (场景化入口)
   - 制作界面: 加速制作
   - 升级界面: 半价升级
   - 顾客界面: 满意度恢复

3. 激励视频
   - 每日奖励: 连续观看
   - 免费材料: 限时领取
   - 特殊顾客: 广告召唤
```

### 6.2 变现策略
```
轻度玩家:
- 看广告加速游戏进度
- 每日签到奖励
- 离线收益双倍

中度玩家:
- 小额充值解锁功能
- 月卡特权
- 限时礼包

重度玩家:
- 大额充值折扣
- 专属装饰
- 提前体验内容
```

## 七、技术实现建议

### 7.1 性能优化
```
1. 加载优化
   - 首屏资源 < 2MB
   - 懒加载非关键资源
   - 资源压缩 (TinyPNG)

2. 渲染优化
   - 使用Canvas 2D而非WebGL
   - 减少透明图层叠加
   - 合理使用缓存

3. 内存管理
   - 及时销毁不可见对象
   - 对象池复用
   - 避免内存泄漏
```

### 7.2 适配方案
```
1. 屏幕适配
   - 750×1334设计稿
   - 等比例缩放
   - 安全区域处理

2. 机型覆盖
   - 支持iOS/Android主流机
   - 测试低端机性能
   - 横竖屏锁定
```

## 总结

本设计在保证微信小游戏性能要求的前提下，通过：
1. **简约界面布局** - 减少操作层级
2. **微恐视觉元素** - 营造氛围但不吓人
3. **变现入口突出** - 关键位置显眼设计
4. **资源控制严格** - 总资源包<10MB

可实现快速开发上线，预计总开发周期4-6周。
