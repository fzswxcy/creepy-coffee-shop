## ⚠️ 防Token溢出保护机制

### 已实现保护
1. **长文本截断**: 自动识别超长内容并进行摘要提取
2. **后台处理**: 爬取逻辑在后台执行，不直接输出到对话窗口
3. **本地日志**: 所有原始数据保存到专用日志文件
4. **精简报告**: 只输出关键摘要和分析结论

### 保护流程
```
爬虫请求 → 获取原始数据 → 后台截断摘要 → 保存到本地日志 → 
生成精简报告 → 输出到对话窗口
```

### 关键配置
- **日志文件**: `/projects/finance_data_log.txt`
- **摘要长度**: 不超过500字符
- **截断阈值**: 1000字符自动触发
- **保护状态**: ✅ 已启用

## 🎯 任务调度原则

### 并行管理规则
1. **线程隔离**: 每个项目在独立线程中运行
2. **状态独立**: 项目状态独立保存，互不影响
3. **工具专用**: 项目工具不交叉使用
4. **内存隔离**: 避免任务间内存污染

### 指令响应逻辑
```
收到指令 → 识别项目类型 → 切换到对应线程 → 
读取项目状态 → 执行具体操作 → 更新状态文件
```

### 切换示例
- **"继续游戏开发"** → 切换到minigame_thread
